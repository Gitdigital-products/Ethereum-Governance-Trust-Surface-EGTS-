# Contributors Wall

## Purpose
This document defines the “Contributors Wall” — a recognition page specifically for code and documentation contributors (excluding financial contributions). It celebrates the project’s developer community.

## Definitions
- **Contributor**: Anyone who has had a pull request merged into any ApexOS repository or has submitted a non‑trivial documentation patch.
- **First‑time Contributor**: A contributor whose first merged PR occurred within the last 90 days.
- **Regular Contributor**: A contributor with 5+ merged PRs in the past year.

## Tiers (if applicable)
Tiers based on number of merged PRs (all time):
- Newcomer: 1 PR
- Active: 5–24 PRs
- Core: 25–99 PRs
- Master: 100+ PRs

## Benefits
- Newcomer: Name on wall, welcome badge.
- Active: Name on wall with GitHub avatar, “Active Contributor” role.
- Core: Avatar, “Core Contributor” badge, priority review for own PRs.
- Master: Avatar, “Master Contributor” badge, one free ticket to any ApexOS event.

## Requirements
- Contributors must have signed the Contributor License Agreement (CLA).
- Contributors must not have had any PR reverted due to malicious code.
- Name appears as GitHub username unless a different name is requested.

## Governance alignment
The Contributors Wall is governed by the Technical Steering Committee. Disputes about contributor status are resolved by the committee. The wall is intended to motivate participation, not to exclude.

## Branding rules
- The wall uses a grid of avatars; hover shows name and PR count.
- No external logos or links except to GitHub profiles.
- The wall must be accessible from the project’s GitHub repository README via a badge.

## Public recognition rules
- New contributors are announced on the `#contributors` Discord channel each week.
- Monthly “Contributor Spotlight” picks one contributor from each tier.
- Contributors Wall is archived yearly and stored in the repository under `/archive/contributors/`.

## Templates (if applicable)
- CLA signature form: embedded via CLA assistant.
- Contributor spotlight interview template: `/templates/spotlight_interview.md`
