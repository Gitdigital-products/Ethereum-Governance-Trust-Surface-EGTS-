=== FILE: contribution_guidelines.md ===

OWNER: GitDigital Products

AUTHOR: RickCreator87

Purpose

Provides instructions for making contributions to EGTS, including pull request process, coding standards, and governance expectations.

Definitions

· Contribution: Any change submitted via pull request, issue, or governance proposal.

Roles & Responsibilities

· GitDigital Products – Enforces guidelines.
· RickCreator87 – Answers questions about guidelines.

Requirements

· All contributions must adhere to the Developer Certificate of Origin (DCO).
· Pull requests must pass CI and receive one approval.
· Governance proposals must follow the template in GOVERNANCE.md.

Contribution Workflow

1. Fork repository.
2. Create a feature branch.
3. Write changes with tests/docs.
4. Submit pull request.
5. Respond to reviewer feedback.
6. Merge after approvals.

Governance Alignment

These guidelines are derived from GOVERNANCE.md and the project’s code of conduct. Violations may result in contribution privileges being suspended.
