Purpose

Defines the quality assurance (QA) processes, standards, and metrics for the EGTS project.

Definitions

· QA: Systematic activities to ensure that deliverables meet quality thresholds.

Roles & Responsibilities

· GitDigital Products – Allocates QA resources.
· RickCreator87 – Performs QA checks and reports metrics.

Requirements

· QA includes automated linting, spell check, and link validation for documentation.
· Code changes require unit test coverage >80%.
· Any QA failure blocks merge.

QA Metrics (current)

· Governance file linting: PASS
· Spelling errors: 0
· Broken links: 0
· Placeholder count: 0

Governance Alignment

QA gates are enforced by CI pipelines as mandated by GOVERNANCE.md.
